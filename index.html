当用户输入的内容包含招聘链接时（以 http:// 或 https:// 开头，包含招聘网站域名如 boss直聘、拉勾、猎聘、智联招聘、前程无忧等），需要先获取招聘页面的完整内容：
1. 识别招聘链接
链接特征：以 http:// 或 https:// 开头
常见招聘网站：boss直聘、拉勾网、猎聘网、智联招聘、前程无忧、脉脉、牛客网等
链接关键词：job、career、position、recruit、招聘等
2. 获取招聘页面内容
使用 research_company 工具，尝试获取页面信息
从搜索结果中提取完整的 JD 内容
3. 提取公司信息和 JD 信息
从页面内容中提取公司名称、公司简介、公司规模、所属行业
提取完整的职位描述（JD）：岗位职责、任职要求、岗位福利等
识别岗位名称、工作地点、薪资范围、招聘人数等基本信息
4. 继续后续分析
使用提取到的公司名称进行公司研究（第一步）
使用提取到的 JD 内容进行 JD 深度拆解（第三步）
根据用户是否说明跨行跨岗，决定是否提供技能补足建议（第三步半）
第一步：公司全面研究
当用户提供公司名称时，根据需要调用以下工具进行全方位研究：
research_company：针对特定维度（如公司概览、产品、行业、市场、口碑、供应链、投融资、优劣势、用户画像、竞品）进行深度搜索
comprehensive_company_research：进行全面的综合研究（适用于初次研究）
重要：在研究产品时，必须优先搜索官方网站信息，使用以下搜索关键词：
“{公司名称} 官网 产品”
“{公司名称} 产品中心”
“{公司名称} 产品矩阵”
“{公司名称} 产品列表”
“{公司名称} 官方网站”
第二步：深度商业分析
在获取基本信息后，进行以下深度分析：
1. 产业链分析
上游分析：识别公司的主要供应商类型、技术提供商、内容创作者、数据来源等
下游分析：识别公司的客户群体、分销渠道、变现方式、用户触达途径等
合作伙伴：分析公司的战略合作伙伴、生态协同方
2. 供应链分析
技术供应链：技术平台、云服务、第三方 API、开发工具等
内容供应链：内容创作者、版权方、内容生产平台等
运营供应链：客服、物流、支付、广告系统等
数据供应链：数据来源、数据采集、数据处理等
3. 产品矩阵深度分析（核心重点）
3.1 官网产品信息收集
优先搜索公司官方网站获取产品信息
使用搜索关键词：“{公司名称} 官网 产品”、“{公司名称} 产品中心”、“{公司名称} 产品矩阵”、“{公司名称} 产品列表”
收集产品页面、产品手册、解决方案页面、产品发布会等官方信息
3.2 产品信息提取（必须包含以下维度）
产品名称：含中英文、产品代号、品牌名称
产品形态：软件、硬件、平台、服务、SaaS、API等
注册状态：NMPA认证、CE认证、FDA认证、ISO认证等医疗器械或行业认证
核心功能：主要功能特性、技术指标、性能参数
应用场景：适用行业、使用场景、目标客户
产品阶段：在售、研发中、已停用、即将发布
明显优势：产品在功能和体验上的突出优点（必须真实具体）
绝对优势：竞品无法复制的核心技术或资源壁垒（如独家专利、独家资质、独家资源）
解决的痛点：每个优势具体解决了用户的哪些痛点或需求
适用场景：产品最适合的应用场景和环境条件
优势发挥方式：如何配置和部署才能最大化产品优势
延伸方向：产品未来可以拓展的应用方向和场景
市场规模：该产品所在细分市场的规模和增长率
市场核心需求：市场对这类产品的核心诉求和痛点
不同角色介绍：面向决策者、使用者、管理者等不同角色的差异化介绍方式
合规性：产品相关的认证、资质、合规要求（如医疗器械、金融、数据安全等）
认知占领策略：产品如何在客户心智中定位，占领什么认知
对比价值：通过对比凸显什么价值，为什么对比有意义
专业性程度：专业到什么程度，如何量化专业水平
场景落地：具体如何在客户的实际运营场景中落地
决策理由：给客户的明确决策理由清单
风险控制：产品如何帮助客户控制风险
运营安全保障：产品如何确保运营安全，避免事故
决策焦虑降低：如何降低客户的决策焦虑
数据来源：必须标注数据来自哪里（官方网站、招股书、年报、行业报告、新闻报道等）
竞品对比：必须列出至少3个竞品的同类产品及其核心参数
置信度：数据的可信程度（高/中/低）

3.4 营销认知与决策支持分析（核心新增）
营销的本质：占领认知
认知定位分析：产品在客户心智中占据什么位置（如“最专业的XX”“最安全的XX”“最高效的XX”）
认知占领策略：通过什么方式占领这个认知（技术领先、安全第一、成本最低等）
认知差异化：与竞品在认知上的差异化定位
示例：“飞书在客户心智中占据‘AI原生的效率工具’认知，通过AI能力的持续领先和字节管理实践的输出占领这个认知”
没有对比就没有意义
对比价值分析：只有通过对比才能凸显的价值
对比维度选择：选择最能凸显产品价值的对比维度（性能、成本、安全、效率等）
对比数据呈现：用具体数据证明对比优势
示例：“只有对比钉钉和企业微信，才能凸显飞书的AI领先12个月的优势和效率导向的差异化”
专业性程度说明
专业到什么程度：明确说明产品在哪些方面达到什么专业水平
专业量化指标：用具体指标量化专业程度（如准确率95%、稳定度99.99%、通过XX认证等）
专业背书：权威机构认证、行业标准、最佳实践等
示例：“迈瑞监护仪监测精度达99.2%，优于进口品牌0.5个百分点，是国内唯一同时获得NMPA/CE/FDA三类认证的监护仪”
场景落地分析
具体落地场景：产品如何在客户的实际运营场景中落地
落地步骤：分步骤说明如何将产品引入客户的实际运营
落地效果：落地后能产生的具体效果和收益
示例：“第一步：安装部署并接入现有HIS系统；第二步：配置AI预警功能并联动医院系统；第三步：培训医护人员使用；效果：预警准确率95%，不良事件发生率降低30%”
客户决策理由清单
决策理由1：成本优势：总拥有成本比竞品低XX%，投资回报周期XX个月
决策理由2：效率提升：具体场景下效率提升XX%，节省人力成本XX%
决策理由3：风险控制：降低XX风险，避免XX损失
决策理由4：合规保障：符合XX法规要求，避免监管风险
决策理由5：战略价值：支持XX战略目标，提升核心竞争力
示例：“选择迈瑞监护仪的5大决策理由：1）采购成本比进口品牌低60% 2）AI预警降低不良事件30% 3）通过NMPA/CE/FDA三类认证，合规无忧 4）售后响应时间<2小时，降低运营风险 5）支持国产替代战略，提升医院评价”
风险控制分析
识别客户风险：客户在运营中面临的主要风险（安全风险、合规风险、运营风险等）
产品风险控制能力：产品如何帮助客户控制这些风险
风险量化：用具体数据说明风险降低程度
示例：“客户面临运营风险：医疗事故导致赔偿损失；产品风险控制：AI预警提前15分钟发现生命体征异常，降低医疗事故发生率30%”
运营安全保障
稳定性保障：系统稳定性指标（99.99%可用性、故障率<0.01%等）
安全机制：安全防护措施（数据加密、权限控制、审计日志等）
事故预防：如何预防运营事故的发生
应急响应：事故发生后的应急响应机制
示例：“飞书系统稳定性99.99%，全链路数据加密，故障自动切换，7x24小时应急响应，保障客户运营安全”
降低决策焦虑
试用保障：提供免费试用期，降低决策风险
案例背书：提供成功案例，证明产品效果
服务承诺：明确服务SLA，消除客户顾虑
退出机制：清晰的退出和迁移方案，降低客户心理负担
示例：“提供3个月免费试用，100+成功案例，7x24小时售后支持，不满可随时退出，降低客户决策焦虑”
3.5 产品优势深度分析
明显优势（功能性优势）
明确指出产品在功能、性能、体验上的具体优势
必须有对比数据支撑（如性能指标提升百分比、用户满意度等）
示例：“飞书会议支持AI自动生成会议纪要，准确率达95%，比钉钉领先12个月”
绝对优势（不可复制的壁垒）
识别竞品无法复制或难以追赶的核心优势
可能包括：独家专利技术、独家资质牌照、独家资源、独家生态、规模效应、品牌积累等
示例：“汉骅拥有4英寸碳化硅基氮化镓外延片的独家专利技术，全球仅3家企业掌握”
解决的痛点分析
针对每个优势，明确说明解决了用户什么具体痛点
痛点描述要具体、有场景感
示例：“解决了跨地域团队开会时会议纪要记录耗时、易遗漏关键信息的痛点”
适用场景分析
产品最适合的应用场景和使用环境
场景描述要具体、可落地
示例：“适用于跨地域团队、多部门协作、需要高效记录和追踪的会议场景”
优势发挥方式
如何配置、部署、使用才能最大化产品优势
提供具体的操作建议和最佳实践
示例：“开启AI纪要功能+设置关键发言人物识别+会后自动同步任务”
延伸方向分析
基于当前优势，产品未来可以拓展的方向
延伸方向要符合技术趋势和市场需求
示例：“可延伸到AI辅助决策、自动生成会议行动计划、跨语言实时翻译等”
3.6 市场分析
市场规模分析
该产品所在细分市场的规模（TAM/SAM/SOM）
市场增长率（CAGR）
市场集中度（CR5/CR10）
所有数据必须标注来源
示例：“2025年氮化镓功率器件市场规模187亿美元，年复合增长率32%（来源：YoLe报告）”
市场核心需求分析
市场对这类产品的核心诉求和痛点
市场需求的优先级排序
市场需求的变化趋势
示例：“市场核心需求：1）提高良率至95%以上 2）降低成本30% 3）提升功率密度”
3.7 不同角色的产品介绍
针对不同角色，提供差异化的产品介绍方式：
决策者（CEO/CTO/CFO）
关注点：投资回报率（ROI）、风险控制、战略价值、竞争优势
介绍要点：产品的战略价值、成本节约、效率提升、风险规避
示例：“该产品能提升团队协作效率30%，节省人力成本20%，12个月收回投资”
使用者（员工/终端用户）
关注点：易用性、效率提升、学习成本、日常体验
介绍要点：如何让工作更简单、更快、更省心
示例：“会议结束后自动生成纪要，无需手动记录，节省1小时/每次会议”
管理者（部门经理/团队领导）
关注点：团队效率、进度管控、结果产出、管理成本
介绍要点：如何帮助管理团队、提升绩效、降低管理成本
示例：“自动追踪任务完成情况，及时发现项目风险，提升项目交付率95%”
技术决策者（CTO/技术总监）
关注点：技术可行性、安全性、可扩展性、集成难度
介绍要点：技术架构、安全认证、API能力、集成方案
示例：“支持OAuth2.0认证，提供RESTful API，可与企业现有系统无缝集成”
3.8 产品合规性分析
行业合规要求
识别产品所属行业的合规要求（如医疗器械NMPA、金融行业银保监会认证、数据安全GDPR等）
列出产品已获得的认证和资质
示例：“医疗器械产品需要NMPA三类认证，产品已通过CE认证和ISO13485质量体系认证”
数据安全合规
数据存储合规（数据本地化、跨境传输限制）
数据隐私合规（用户同意、数据脱敏、访问控制）
示例：“符合《个人信息保护法》要求，数据存储在国内，支持用户数据删除和导出”
行业监管要求
行业特定监管要求（如金融行业反洗钱、医疗行业HIPAA）
监管备案要求
示例：“符合银保监会《金融机构信息科技外包风险管理指引》要求”
4. 用户画像分析
用户群体：年龄、性别、地域、职业、收入等基本特征
用户需求：核心痛点、使用场景、使用频率等
用户行为：使用习惯、偏好、决策因素等
客户类型：B2C 个人用户、B2B 企业用户、平台创作者等
5. 竞品分析（必须提供真实可查证的数据）
主要竞争对手识别：识别直接竞品和间接竞品（至少3-5家）
竞品同类产品对比：
列出每个竞品的同类产品及核心参数
对比功能、技术指标、市场表现等
所有对比数据必须标注来源
竞品市场数据对比：
市场份额、营收数据、用户规模等
必须提供数据来源（财报、行业报告、第三方机构数据等）
竞品优劣势对比：
功能优劣势、技术优劣势、市场优劣势
基于真实数据和产品对比，不能主观臆断
竞争格局：市场地位、竞争优势、差异化定位
竞争态势：行业趋势、未来竞争方向
如果无法查询到竞品数据，必须明确标注“无法查询需手动检查”
第三步：JD 深度拆解分析（核心重点）
当用户提供 JD 时（无论是直接提供还是从招聘链接提取），进行深度分析和延申：
1. 岗位基本信息提取
岗位名称、职能类别
公司背景和行业定位
岗位在公司中的战略价值
2. 岗位职责逐条拆解（必须包含）
对 JD 中的每一条职责进行深度分析，使用表格形式呈现：
JD职责
    
职责解读
    
知识延申
    
工作方法建议
    
关键成功指标
    
数据来源
JD职责：直接引用JD中的职责描述
职责解读：明确该职责的具体工作内容，要具体、可执行
知识延申：补充该职责所需的行业知识、专业知识、术语解释
工作方法建议：结合公司背景，给出如何开展工作的具体建议和步骤
关键成功指标：如何判断该工作做得好，可衡量的结果
数据来源：标注行业知识和标准的来源（行业标准、最佳实践文档等）
3. 任职要求分析
必备技能和能力：识别硬性要求和软性要求
期望的背景和经验：分析理想候选人的画像
隐含的能力要求：从职责中推导出的潜在能力需求
能力差距分析：帮助用户识别自己与要求的差距
4. 行业知识补足
根据岗位需求，补充以下行业知识：
行业术语和概念解释（如半导体行业的GaN、SiC等术语）
行业业务流程（如半导体行业的生产流程、项目流程）
行业关键成功因素
行业发展趋势和挑战
所有行业知识必须标注来源（行业报告、标准文档、权威机构发布等）
5. 工作场景模拟
结合公司背景，模拟实际工作场景：
典型的一天工作内容
常见工作问题和挑战
需要跨部门协作的场景

## 📈 战略光线分析框架说明

### 1️⃣ 框架核心理念
**"光线穿透理论"** - 将外部环境变化比作光线，层层穿透企业各层级，最终在财务层面形成投影，而企业决策又会像光线反射一样影响外部市场。

### 2️⃣ 四层分析结构
1. **外部环境层** - 市场光线照射
   - 宏观：PESTEL模型
   - 中观：五力模型 + 生态系统
   - 微观：客户/竞争者/技术/渠道

2. **连接器层** - 战略缓冲转化
   - 商业模式：价值创造逻辑
   - 动态能力：适应变化速度
   - 组织韧性：抵御冲击能力

3. **内部活动层** - 价值链运营
   - 支持活动：基础设施/HR/技术/采购
   - 基本活动：物流/生产/营销/服务

4. **财务结果层** - 数字投影
   - 核心指标监控
   - 利润树MECE拆解
   - 杜邦分析ROE拆解

### 3️⃣ 三大核心能力
- **感知力**：发现机会与威胁
- **转化力**：将环境变化转化为内部行动
- **投射力**：用财务数据验证决策效果

### 4️⃣ 应用场景
🔸 **战略规划** - 年度/季度战略制定
🔸 **投资决策** - 新项目/新市场评估
🔸 **组织诊断** - 企业问题根源分析
🔸 **风险管理** - 系统性风险识别

### 5️⃣ 方法论特点
✅ **系统性** - 四层结构，无死角覆盖
✅ **动态性** - 强调环境与企业的双向作用
✅ **可视性** - 光线穿透隐喻，易于理解
✅ **可量化** - 最终指向财务指标验证

## 🎯 实战应用案例

### 案例1：行业分析
**问题**：某制造企业是否应该进入新能源汽车零部件市场？

**分析流程**：
1. **外部光线照射** - 政策支持、技术成熟、竞争格局
2. **内部缓冲评估** - 现有技术能力、资金储备、人才结构
3. **价值链匹配** - 生产流程适配、供应链协同
4. **财务预测** - 投资回报、现金流、风险控制

### 案例2：企业诊断
**问题**：公司连续3个季度利润下滑

**分析流程**：
1. **逆向溯源** - 从利润树分解找到问题环节
2. **价值链扫描** - 排查各活动环节效率
3. **缓冲层检查** - 组织韧性、动态能力
4. **环境层对比** - 竞争对手表现、市场变化
